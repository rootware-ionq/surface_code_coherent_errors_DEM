## Analysis Notebooks
These are handwritten notebooks to test the surface code simulation outputs. Each analyses three separate tests done in `/src/main.rs`:

- `First Step : building the DEM.ipynb`: This file reads in `x_ancilla_probs.csv` which contains the simulation results for
  - preparing the initial state $`\ket{+}_L`$
  - adding the same coherent error ($`Z`$ error with $`\theta = 0.1\pi`$) to all data qubits
  and constructs the DEM decoder graph by inferring the $`p_{ij}`$ error probabilities from the ancilla measurements. Finally, it checks if the reconstructed angle $`\tilde{\theta}`$ from the error probabilities matches expected based on applied $`\theta`$.
- `Second Step: testing math from Notes.ipynb`: This file tests the hypothesis that if the initial state is in the -1 eigenspace of stabilizer $`Z_2Z_5`$, then the error for the boundary DEM edge corresponding to data qubits $2$ and $5$ will have coherent errors _cancel_. It reads in the ancilla measurements from `x_ancilla_probs_2.csv`.
- `Third Step: looking at non uniform coherent errors.ipynb`: Using the $`\ket{+}_L`$ test what happens when we have $Z$ coherent errors with different $\theta$ applied to each data qubit. In particular, we test a constructed case where the coherent errors on data qubits 2 and 5 (corresponding to the boundary DEM edge for detector 1) are equal in magnitude and opposite in sign $`\theta_{2}=`\theta_5`$, and the coherent errors on data qubits for detector 2 are equal in both magnitude and sign $`\theta_{4}-\theta_6`$. We notice that this leads to $`p_{11}=0$, and $`p_{22}=\sin^2\tilde{\theta}`$ with $`\tilde{\theta}= (\theta_{4}+\theta_6})`$. This reads in the ancilla measurements from `x_ancilla_probs_3.csv`, which is generated by doing the corresponding test for the surface code.
